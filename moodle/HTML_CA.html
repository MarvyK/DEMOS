HTML and CSS - Course Assignment Instructions


Introduction

In this assignment, use your newly acquired HTML and CSS skills to transform your Rainy Days design (from the Design Course) into a working reality.


Learning Outcomes

  • You are proficient in HTML and CSS, including semantic markup and styling, and can develop functional and user-friendly web solutions.
  • You are skilled in responsive web design, utilizing CSS media queries, fluid layouts, and adaptive techniques to ensure website functionality across various devices and screen sizes, and you leverage DevTools for cross-platform and cross-device testing.
  • You ensure website quality and accessibility by using validation tools to meet standards, applying WCAG guidelines, and conducting user testing to identify and fix visual and technical issues.
  • You understand and apply ethical principles of universal design in web development, ensuring accessibility and inclusivity in your HTML and CSS websites while adhering to relevant regulations, guidelines, and the WCAG standard.

Take note of the following

  • Ensure that all your images are compressed to around 200KB.
  • The HTML should be semantic, neatly formatted and valid.
  • The CSS should follow the DRY principles and be easy to read.
  • The site should be WCAG compliant, SEO, performance and accessibility should be considered.
  • Each page should have a unique <meta name="description">, <title> and <h1>.

Grading criteria (Grading scale - Pass / Fail)

  1. Responsiveness:
    • The website is responsive across screen sizes.
    • The website works on major browsers and across major devices.
  2. Bug-free, neat and well written code:
    • Each page has a unique title.
    • Each page has a unique meta description.
    • Semantic HTML5 elements are used where appropriate.
    • The HTML code is bug-free.
    • No unnecessary inline styling.
    • The CSS follows DRY principles.
    • The HTML and CSS are formatted neatly.
    • Images are within the 200kb recommended file size to ensure faster loading time.
  3. Appeal of website and ease of navigation:
    • The website has no broken links.
    • Navigation between pages is clear.
    • The website appeals to the target audience.
  4. WCAG friendly:
    • Images have alt text.
    • Ensure sufficient color contrast.
    • Semantic HTML is used to ensure accessibility with screen readers
