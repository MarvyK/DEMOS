Project Methodology Course Assignment


Introduction

Leverage your experience designing and building an HTML website to form a flexible project plan for your upcoming semester project examination.

The semester project examination [SP1] involves planning for the design of the website [design assets will be supplied], development environment setup, the coding of the website, the deployment and delivery thereof and a project report.


Learning Outcomes

  • You have knowledge of organizational requirements and workflows in relation to web development and agile project methods
  • You have knowledge of processes and tools that are used in project management and documentation.
  • You are able to use relevant tools and techniques for project planning, process documentation and project completion.
  • You have knowledge of concepts and processes that are used for analysis, project planning, completion, reports and presentations
  • You can apply knowledge of project methodologies for efficient project completion.
  • You can find information and material that is relevant for project development with agile project methods.
  • You can carry out work with files and file systems and develop methods to ensure lean workflow in a development project.

Brief

Using GitHub Projects, create a Kanban and Gantt style chart to serve as a living project plan that you can use to track and communicate your progress. Although you do not have the exam brief, your plan should still provide a useful structure by specifying each step in the process.
Use the Progression Plan spreadsheet to assist with selecting dates for each phase or task.

Take note of the following

  • There should be a minimum of 15 discrete tasks in your plan. If you have fewer than this your task definitions may be too broad and should be made more specific. Process

  1. Create a new public GitHub Projects board with the name `PM Project`.

  2. Use the Kanban template to create a Kanban view with the appropriate status columns. See template below:


  3. Within your Project page create a new Roadmap view tab.

  4. Within your Roadmap view, create two new Date fields called Start Date and Target Date.

  5. Create a task/issue for each step in your project plan with the following details:

    • Title (A short description of the task)
    • Description (A detailed description of the task)
    • Size (How much effort the task will require)
    • Start Date (When the task should be started)
    • Target/End Date (When the task should be completed)
